# PM-Hub — Claude Skills for Product Managers

专为产品经理设计的 Claude Skill 合集，覆盖从需求文档、竞品分析到 Web3 项目管理的核心 PM 工作流。

---

## Skills 列表

| Skill 文件 | 功能 | 触发关键词 |
|---|---|---|
| `prd-writer.skill` | 自动生成结构规范的产品需求文档（PRD） | "帮我写PRD"、"写需求文档" |
| `competitive-analysis.skill` | 自动生成竞品分析报告 | "帮我做竞品分析"、"分析竞品" |
| `Web3Project.skill` | 自动生成 Web3 项目分析报告 | "帮我做Web3项目分析"、"分析这个Web3项目" |

---

## 安装方法

1. 下载对应的 `.skill` 文件
2. 在 Claude 桌面应用中，点击左侧边栏的 **Skills** 入口
3. 点击 **Install Skill**，选择下载好的 `.skill` 文件完成安装
4. 安装后，当你在对话中提到对应关键词时，Claude 会自动调用该 Skill。

## Skill 1：PRD Writer

### 简介

PRD Writer 是一个专为产品经理设计的 Claude Skill，能根据你的自然语言描述，自动生成结构规范、格式统一的产品需求文档（PRD）。输出为可在浏览器直接打开的 **HTML 文件**，内含可视化 Mermaid 流程图、线框原型图和需求明细表格。

### 触发方式

以下任意描述均可触发 PRD Writer：

- "帮我写一个 XXX 功能的 PRD"
- "写需求文档，功能是……"
- "我需要一份产品需求，内容是……"
- 直接描述一个功能需求（即使没有说 "PRD" 也会触发）

### 输出内容

每份 PRD 包含以下章节：

| 章节 | 内容 |
|---|---|
| 0 更改日志 | 版本记录表格（日期 / 版本号 / PM / 修改内容） |
| 1 背景 | 用户痛点、产品目标、风险说明 |
| 2 相关材料 | 原型图、设计稿、竞品参考等链接占位 |
| 3 流程 | 角色定义 + 中文 Mermaid 流程图 |
| 4 需求 | 需求模块总览 + 每个子功能的线框原型图 + 需求明细表格 |
| 4.4.X 计算汇总 | 变量定义、计算公式、数字举例（如有计算逻辑） |
| 5 其他 | 边缘 Case、风险、灰度方案等 |
| 6 监控 | 关键数据指标与监控日报格式 |

输出文件：

- `prd.html` — 可直接在浏览器打开，流程图和原型图均可渲染
- `prd.md` — 原始 Markdown 文件，方便后续编辑

### 使用示例

**输入：**

> 帮我写一个现货账户小额资产兑换为平台币的 PRD，目标用户是散户

**输出：**

完整 HTML PRD，包含用户流程图、各页面线框原型（正常态 + 空状态）、需求明细表格、计算公式举例和监控指标。

## Skill 2：Competitive Analysis

### 简介

Competitive Analysis 是一个专为产品经理设计的 Claude Skill，能根据你提供的产品方向或竞品名称，自动生成结构化的竞品分析报告。输出涵盖竞品概览、核心功能对比、差异化亮点、用户评价摘要及战略建议，帮助你快速掌握市场格局。

### 触发方式

以下任意描述均可触发 Competitive Analysis：

- "帮我做一份 XXX 产品的竞品分析"
- "分析一下 XXX 和 YYY 的竞品对比"
- "我要做竞品研究，产品是……"
- 直接描述你的产品方向并要求了解市场现状

### 输出内容

每份竞品分析报告包含以下模块：

| 模块 | 内容 |
|---|---|
| 竞品概览 | 各竞品基本信息、定位、目标用户 |
| 功能对比矩阵 | 核心功能逐项对比表格 |
| 差异化亮点 | 各竞品的核心竞争力与独特卖点 |
| 用户评价摘要 | 用户反馈中的高频痛点与正面评价 |
| SWOT 分析 | 主要竞品的优势、劣势、机会、威胁 |
| 战略建议 | 基于分析的产品差异化切入点建议 |

### 使用示例

**输入：**

> 帮我分析一下国内主流的 AI 笔记产品竞品，包括 Notion AI、飞书文档 AI、印象笔记

**输出：**

结构化竞品分析报告，包含功能对比矩阵、各产品差异化亮点、用户痛点摘要及产品策略建议。

## Skill 3：Web3 Project Analysis

### 简介

Web3 Project Analysis 是一个专为产品经理设计的 Claude Skill，能根据你提供的 Web3 项目名称或方向，自动生成结构化的项目分析报告。输出涵盖项目概览、代币经济模型、技术架构评估、链上数据分析及投资风险判断，帮助你快速理解 Web3 项目的核心价值与潜在风险。

### 触发方式

以下任意描述均可触发 Web3 Project Analysis：

- "帮我分析一下 XXX 这个 Web3 项目"
- "做一份 XXX 的 Web3 项目报告"
- "我想了解 XXX 项目的代币经济模型"
- 直接描述一个区块链 / DeFi / NFT / GameFi 项目并要求分析

### 输出内容

每份 Web3 项目分析报告包含以下模块：

| 模块 | 内容 |
|---|---|
| 项目概览 | 项目背景、赛道定位、核心团队 |
| 代币经济模型 | 代币分配、解锁计划、激励机制 |
| 技术架构 | 底层公链、合约安全性、技术创新点 |
| 链上数据分析 | TVL、活跃地址、交易量等关键指标 |
| 竞品对比 | 同赛道项目横向对比 |
| 风险评估 | 监管风险、流动性风险、合约风险等 |
| 投资建议 | 基于分析的结论与注意事项 |

### 使用示例

**输入：**

> 帮我分析一下 Uniswap V4 这个 DeFi 项目，重点看代币经济模型和竞品对比

**输出：**

结构化 Web3 项目分析报告，包含 UNI 代币经济模型、与 Curve / Balancer 的功能对比、链上核心指标及风险评估。

---

## 注意事项

- 如果你没有提供足够的背景信息，Skill 会用 `【待确认：XXX】` 占位，不会中断生成，之后补充即可。
- 中英文混用，专业术语保留英文（如 PRD、MVP、DAU、Retention、TVL、DeFi 等），符合互联网产品团队的写作习惯。
- 如需英文输出，可在描述中注明"请用英文输出"。

## 常见问题

**Q: 如何同时使用多个 Skill？**
A: 每个 `.skill` 文件单独安装即可，Claude 会根据对话内容自动判断调用哪个 Skill。

**Q: Skill 支持哪些版本的 Claude 桌面应用？**
A: 建议使用最新版本的 Claude 桌面应用以获得最佳体验。

**Q: 生成的内容可以修改吗？**
A: 可以。生成后直接编辑输出文件，或在对话中告诉 Claude 需要调整的部分，Claude 会帮你修改。

**Q: Web3 Project Analysis 的链上数据从哪里来？**
A: Claude 会基于自身知识库进行分析。如需最新实时数据，建议结合 DeFiLlama、Dune Analytics 等工具补充。

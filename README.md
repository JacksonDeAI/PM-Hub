# PRD Writer Skill — 使用说明

## 简介

PRD Writer 是一个专为产品经理设计的 Claude Skill，能根据你的自然语言描述，自动生成结构规范、格式统一的产品需求文档（PRD）。输出为可在浏览器直接打开的 **HTML 文件**，内含可视化 Mermaid 流程图、线框原型图和需求明细表格。

---

## 安装方法

1. 下载 `prd-writer.skill` 文件
2. 在 Claude 桌面应用中，点击左侧边栏的 **Skills** 入口
3. 点击 **Install Skill**，选择下载好的 `.skill` 文件完成安装

安装后，当你在对话中提到"写PRD"、"帮我写需求文档"等关键词时，Claude 会自动调用该 Skill。

---

## 触发方式

以下任意描述均可触发 PRD Writer：

- "帮我写一个 XXX 功能的 PRD"
- "写需求文档，功能是……"
- "我需要一份产品需求，内容是……"
- 直接描述一个功能需求（即使没有说"PRD"也会触发）

---

## 输出内容

每份 PRD 包含以下章节：

| 章节 | 内容 |
|------|------|
| 0 更改日志 | 版本记录表格（日期 / 版本号 / PM / 修改内容） |
| 1 背景 | 用户痛点、产品目标、风险说明 |
| 2 相关材料 | 原型图、设计稿、竞品参考等链接占位 |
| 3 流程 | 角色定义 + 中文 Mermaid 流程图 |
| 4 需求 | 需求模块总览 + 每个子功能的线框原型图 + 需求明细表格 |
| 4.4.X 计算汇总 | 变量定义、计算公式、数字举例（如有计算逻辑） |
| 5 其他 | 边缘 Case、风险、灰度方案等 |
| 6 监控 | 关键数据指标与监控日报格式 |

**输出文件：**
- `prd.html` — 可直接在浏览器打开，流程图和原型图均可渲染
- `prd.md` — 原始 Markdown 文件，方便后续编辑

---

## 使用示例

**输入：**
> 帮我写一个现货账户小额资产兑换为平台币的 PRD，目标用户是散户

**输出：** 完整 HTML PRD，包含用户流程图、各页面线框原型（正常态 + 空状态）、需求明细表格、计算公式举例和监控指标。

---

## 注意事项

- **缺失信息的处理：** 如果你没有提供原型图链接、计算公式等细节，文档会用 `【待确认：XXX】` 或 `[待补充]` 占位，不会中断生成，之后补充即可。
- **PM 名字：** 默认留空为 `[PM 名字]`，可在生成后手动替换。
- **语言风格：** 中英文混用，专业术语保留英文（如 Deposit、APR、TVL、Swap 等），符合互联网产品团队的写作习惯。
- **原型图风格：** 内嵌线框图为黑白灰风格，无彩色，适合需求文档使用。

## 常见问题

**Q: 流程图在 HTML 文件里显示不出来？**
A: 请用浏览器（Chrome / Safari / Edge）打开 `.html` 文件，不要直接在文本编辑器中预览。

**Q: 我想修改生成的 PRD 怎么办？**
A: 直接编辑 `prd.md` 文件后，重新让 Claude 转换为 HTML；或者直接编辑 `prd.html` 中的对应内容。

**Q: 支持英文 PRD 吗？**
A: 当前 Skill 默认以中文为主输出，如需英文版，可在需求描述中注明"请用英文输出"。
